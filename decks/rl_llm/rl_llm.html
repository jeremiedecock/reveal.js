Why do we need a second stage of fine-tuning?

  • Supervised fine-tuning alone cannot guarantee outputs that are fully aligned with human preferences.
  • Collecting larger, high-quality human-labeled datasets is too expensive or impractical.

Preference-based fine-tuning enables direct optimization for human alignment using feedback, without requiring exhaustive annotation → much cheaper.

So, why do we need a second stage of fine-tuning? Mainly for two reasons: - As I already mentioned, the first reason is that supervised fine-tuning alone cannot guarantee outputs that are fully aligned with human preferences - And the second reason is the lack of high-quality labeled datasets to further improve alignment. Preference-based fine-tuning enables direct optimization for human alignment using feedback, without requiring exhaustive annotation. This is much cheaper.

Learn from feedback

A generalization of the RL framework

  • RL agents need feedback to learn preferred behaviors
  • Usually provided by a reward function (maps states/actions to scalar values)
  • Explicit reward functions can be hard to design in real-world tasks
  • Alternative feedback sources (e.g., human preferences, demonstrations) can guide learning
  • This generalizes RL to "learning from feedback"
Training an RL agent requires some information to indicate what behaviors are preferred. This is typically provided by a reward function that maps states and actions to scalar values. However, in some real-world scenarios, designing an explicit reward function is challenging or infeasible. In such cases, we can use other kinds of feedback to guide the learning process. This leads to a generalization of the RL framework called "learning from feedback".

Reward Model

A quick introduction

  • When a reward function is unavailable, we may learn a reward model from trajectory examples.
  • Use the reward model’s output as reward signals for RL algorithms.
  • This approach is known as Inverse Reinforcement Learning (IRL).
  • IRL will be covered in more detail next week.
For the tasks that the reward function is not available, we may consider learning a reward model and then apply the output of this reward model as reward signals on the RL algorithms. In other words, we learn the reward function from examples of desired behavior. This is called Inverse Reinforcement Learning (IRL) (more on that next week).

PbRL: Preference-based RL

Key Idea:

  • Instead of scalar rewards, use preferences over trajectory segments as feedback.
  • Train a reward model to predict which segment is preferred.
  • Use the learned reward model to provide reward signals for RL algorithms.
  • Enables learning from comparative feedback rather than absolute rewards.

👉 When the environment does not provide a reward function, but a preference feedback, we do Preference-based RL (PbRL).

One example of feedback different from scalar rewards is preferences. Preferences can be collected by asking humans to compare pairs of trajectory segments and indicate which one they prefer.

Reinforcement Learning from Human Feedback (RLHF)

  • RLHF first developed by OpenAI (2017) for training robots in simulated environments and Atari games.
  • First, agent acts randomly; human compares pairs of video clips and selects the one closer to the objective (e.g., backflip).
  • Algorithm learns a reward function that matches human preferences.
  • Uses RL to optimize agent behavior toward the learned goal.
  • Continually requests human feedback on uncertain trajectory pairs to refine the reward model.
  • Demonstrates high sample efficiency: backflip task required less than 1000 bits of human feedback.
Originally, RLHF was developed by OpenAI in 2017 for training simple robots in simulated environments and Atari games. Here, the RL agent starts by acting randomly in the environment. Periodically, two video clips of its behavior are given to a human, and the human decides which of the two clips is closest to the objective: in this case, the objective is to do a backflip. The algorithm gradually builds a model of the goal of the task by finding the reward function that best explains the human’s judgments. It then uses RL to learn how to achieve that goal. As its behavior improves, it continues to ask for human feedback on trajectory pairs where it’s most uncertain about which is better, and further refines its understanding of the goal. This approach demonstrates good sample efficiency. The backflip video required under 1000 bits of human feedback.

RLHF

Advantages and drawbacks

Advantage:

  • Compared to alternative preference fine-tuning methods such as Direct Preference Optimization (DPO), RLHF typically requires less human feedback to reach comparable performance.

Drawbacks:

  • Training can be unstable; the Reward Model may be inaccurate, leading to misaligned or unintended behaviors.
  • The PPO policy may exploit Reward Model weakness (reward hacking).
RLHF has one big advantage for the LLMs usecase: compared to alternative preference fine-tuning methods such as Direct Preference Optimization (DPO), RLHF requires less human feedback to achieve similar performance. But it has also one big drawback: it can be unstable : - the Reward Model may be inaccurate, or - the Reward Model may be "exploited" by the policy.

Reasoning-based fine-tuning

Context

  • Reasoning tasks (math, CS, logic) have a single correct answer and can be automatically verified.
  • Unlike creative tasks, no human feedback is needed for evaluation.
  • Enables automatic, scalable assessment of LLM outputs.
  • DeepSeek-R1 applies a third fine-tuning stage for reasoning using RL with verifiable rewards.
There's a special class of NLP problems called reasoning tasks, which includes math, computer science, logic, and so on. Unlike open-ended or creative tasks, reasoning tasks typically have a single correct answer and a deterministic procedure for verifying the model’s output. This property enables automatic evaluation of LLMs on reasoning tasks, without requiring human feedback for each example. DeepSeek-R1 leverages this by applying a third stage of fine-tuning specifically for reasoning, using a Reinforcement Learning framework that exploits verifiable rewards.

Reinforcement Learning with Verifiable Rewards

So here a reinforcement Learning algorithm is used to fin The reasoning task fit well within the RL framework: - the policy is the LLM; it generates a sequence of tokens (actions); here we use a stochastic policy, so it output a probability distribution over the full dictionnary of tokens at each step; - the state is the current context (prompt + generated tokens) - and the reward is 1 if the final answer is correct, 0 otherwise. using a reward signal that is automatically computed based on whether the final answer is correct or not.

Reasoning-based fine-tuning

Group Relative Policy Optimization (GRPO)

The reinforcement Learning algorithm used in DeepSeek-R1 is called Group Relative Policy Optimization algorithm (or GRPO). It is very similar to PPO except that it uses a different advantage estimation method. The critic network is removed and the advantage is computed by normalizing the returns across a group of episodes. That is to say, for each prompt to assess, multiple rollouts are performed, and the advantage is computed by normalizing the returns of these rollouts. Removing the critic network simplifies and fasten the training.
